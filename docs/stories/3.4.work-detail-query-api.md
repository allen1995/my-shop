# Story 3.4: 作品详情查询API

## Status

Draft

## Story

**As a** user,
**I want** to view the details of a specific work,
**so that** I can see all information about it.

## Acceptance Criteria

1. 后端提供 `/api/works/{workId}` GET接口
2. 接口返回作品的完整信息
3. 验证作品属于当前用户
4. 支持作品不存在的情况处理
5. 接口响应时间小于100ms

## Tasks / Subtasks

- [ ] Task 1: 实现作品详情查询接口 (AC: 1, 2, 3, 4)
  - [ ] 在WorkController中实现GET /api/works/{workId}接口
  - [ ] 接收workId路径参数
  - [ ] 从UserContext获取当前用户ID
  - [ ] 查询作品并验证属于当前用户
  - [ ] 如果作品不存在，返回404
  - [ ] 如果作品不属于当前用户，返回403或404
  - [ ] 返回作品完整信息
- [ ] Task 2: 优化查询性能 (AC: 5)
  - [ ] 确保Repository查询使用索引
  - [ ] 验证响应时间小于100ms
- [ ] Task 3: 创建单元测试 (AC: 1, 2, 3, 4)
  - [ ] 创建WorkControllerTest（更新）
  - [ ] 测试查询作品详情
  - [ ] 测试作品不存在情况

### API Specifications

**GET /api/works/{workId}**
- 描述：获取作品详情
- 认证：需要JWT Token
- 路径参数：workId
- 响应：`{ work: Work }`
- 错误：404（作品不存在或不属于当前用户）

[Source: docs/fullstack-architecture.md#API Specification]
