# Story 4.3: 商品详情查询API

## Status

Draft

## Story

**As a** user,
**I want** to view product details,
**so that** I can see all available options (colors, sizes).

## Acceptance Criteria

1. 后端提供 `/api/products/{productId}` GET接口
2. 接口返回商品的完整信息
3. 包含所有可用的颜色和尺寸选项
4. 接口响应时间小于100ms

## Tasks / Subtasks

- [ ] Task 1: 实现商品详情查询接口 (AC: 1, 2, 3)
  - [ ] 在ProductController中实现GET /api/products/{productId}接口
  - [ ] 接收productId路径参数
  - [ ] 查询商品详情
  - [ ] 如果商品不存在，返回404
  - [ ] 返回商品完整信息（包括colors和sizes）
- [ ] Task 2: 优化查询性能 (AC: 4)
  - [ ] 确保Repository查询使用索引
  - [ ] 验证响应时间小于100ms
- [ ] Task 3: 创建单元测试 (AC: 1, 2, 3)
  - [ ] 创建ProductControllerTest（更新）
  - [ ] 测试查询商品详情
  - [ ] 测试商品不存在情况

### API Specifications

**GET /api/products/{productId}**
- 描述：获取商品详情
- 无需认证（公开接口）
- 路径参数：productId
- 响应：`{ product: Product }`
- 错误：404（商品不存在）

[Source: docs/fullstack-architecture.md#API Specification]
