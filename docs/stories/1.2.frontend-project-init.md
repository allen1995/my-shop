# Story 1.2: 前端项目初始化

## Status

Draft

## Story

**As a** developer,
**I want** to set up the uni-app project with uView UI,
**so that** I have a working frontend foundation for the WeChat mini-program.

## Acceptance Criteria

1. uni-app项目已创建，使用Vue3语法
2. uView UI组件库已集成
3. 项目结构符合uni-app规范
4. 基础页面路由已配置
5. 网络请求封装已完成（基于uni.request）
6. 项目能够在微信开发者工具中正常运行
7. 底部Tab导航已配置（首页、作品集、购物车、我的）

## Dev Notes

### Previous Story Insights

这是前端项目的第一个故事，依赖Story 1.1（后端项目初始化）完成后才能进行API联调。

### Data Models

前端使用TypeScript接口定义数据模型，与后端实体类对应。主要模型定义在API服务层。

[Source: docs/fullstack-architecture.md#Data Models]

### API Specifications

网络请求封装需要支持：
- 自动添加JWT Token到请求头
- 统一错误处理
- Token过期自动跳转登录页
- 请求和响应拦截

[Source: docs/fullstack-architecture.md#Components]

### File Locations

基于项目结构，需要创建以下文件：

**前端项目结构：**
```
frontend/
├── pages/
│   ├── index/
│   │   └── index.vue              # 首页
│   ├── works/
│   │   └── list.vue               # 作品集页
│   ├── cart/
│   │   └── cart.vue               # 购物车页
│   └── profile/
│       └── profile.vue            # 个人中心页
├── api/                           # API接口封装（后续创建）
├── store/
│   └── user.js                    # 用户状态管理
├── utils/
│   └── request.js                 # 网络请求封装
├── static/                        # 静态资源
├── App.vue                        # 应用入口
├── main.js                        # 入口文件
├── manifest.json                  # 应用配置
├── pages.json                     # 页面配置
└── package.json                   # 依赖配置
```

[Source: docs/fullstack-architecture.md#Repository Structure]
